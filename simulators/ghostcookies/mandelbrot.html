Shop Types
  • Efficient Shop: Optimal resource allocation with low cost and high efficiency (85-95% efficiency, 0.2-0.3 cost)
  • Inefficient Shop: Clear suboptimal patterns with high cost and low efficiency (30-50% efficiency, 0.6-0.9 cost). May show hidden benefits like buffer capacity.
  • Mandelbrot Shop: Complex fractal-based patterns that appear inefficient but enable predictive optimization and proactive fraud detection
Operational Phases
Phase 1: Operation Generation
  • Generates operations across multiple time periods (cycles)
  • Each operation has cost, efficiency, and resource metrics
  • Operations represent one business cycle in the timeline
Phase 2: Markov Blanket Analysis
  • Analyzes dependency relationships (parents and spouses)
  • Exponential cost stacking: each dependency layer multiplies costs (1.2x, 1.4x, 1.6x)
  • Red lines = high inefficiency, Green lines = low inefficiency
Phase 3: Fractal Pattern Generation
  • Mandelbrot shops generate complex fractal patterns
  • Patterns reveal hidden optimization opportunities
  • Animated points show evolving complexity over time
Phase 4: Metric Calculation
  • Calculates efficiency, cost density, Markov stack height
  • Evaluates fractal dimension and fraud probability
  • Updates in real-time as parameters change
Phase 5: Second-Order Benefit Analysis
  • Identifies hidden benefits of apparent inefficiencies
  • Distinguishes between fraud and strategic complexity
  • Reveals long-term optimization opportunities
Visualization
  • X Axis: Operation Periods (time progression through cycles)
  • Y Axis: Efficiency Score (vertical height)
  • Z Axis: Cost Density (depth perspective)
  • Color: Resource Level (4th dimension - brighter = more resources)
  • Red Lines: High inefficiency in Markov blanket dependencies
  • Green Lines: Low inefficiency connections
  • Animated Points: Fractal complexity patterns (Mandelbrot Shop only)
Parameters & Cycles
Operation Periods: Number of time cycles to analyze (more periods = longer analysis)
Resource Level: Available resources per operation cycle
Fraud Threshold: Sensitivity level for detecting anomalies (lower = more sensitive)
Markov Depth: Dependency layers analyzed (deeper = more complex cost stacking)
Fractal Complexity: Detail level of Mandelbrot patterns (higher = more complex)

Analysis Cycles:
  • Data Generation Cycle: Creates operations, Markov blankets, and fractals
  • Visualization Cycle: Renders 3D surface plots and connections
  • Real-Time Analysis Cycle: Updates metrics and fraud probability continuously
  • Interactive Exploration Cycle: Allows parameter adjustment and comparison
Fraud Detection Mechanisms
The system detects fraud through multiple mechanisms:
  • Exponential Cost Stacking: Identifies when dependencies create exponentially growing costs beyond normal operations
  • Pattern Analysis: Compares actual efficiency against optimal baselines to detect anomalies
  • Fractal Complexity: Uses Mandelbrot calculations to identify hidden inefficiency patterns
  • Second-Order Analysis: Evaluates whether inefficiencies serve hidden purposes vs. true fraud

Key Insight: Not all inefficiencies are fraud—some enable superior long-term optimization through complex pattern recognition.
